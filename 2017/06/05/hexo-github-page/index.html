hexo+github page 搭建个人博客

发表于 2017-06-05 | 更新于: 2017-12-05 | 分类于 技术类 | | 热度: ℃ |
字数统计: 1,200字 | 阅读时长 ≈ 4分钟

为什么选择hexo？

  • hexo是一款基于Node.js的静态博客框架，关键它支持markdown。而我作为一个前端人员，自己就不需要在安装其他麻烦的东西了~

准备工作

  • 前面也说过了，hexo是基于nodejs的，说以我们当然需要在此之前安装nodejs，安装nodejs，如果你只是想要搭建自己的博客，不怎么用node的话，建议你直接去node官网下载安装一个就好了详情可以看看我的另一篇文章【关于nodejs的3M】。
  • 安装git，直接去git官网上去下载就好了。
  • 安装完成后就开始安装hexo了

安装hexo

  • 在任意位置点击右键，选择 git bash here
    输入

  • 创建一个文件夹，名字你认识就好，不过既然是hexo搭建就叫hexo就好了
  • 现在你还可以在里面创建文件夹，单独放你的博客文档，这里就不创建了直接在hexo目录下进行初始化，下面的例子全是在hexo目录下进行的。
  • 和第一步一样，在hexo目录下点击右键，选择git bash here
    输入

耐心等待一段时间，这里的时间比较长，因为hexo正在帮你创建你需要的文件

  • 在hexo目录下，有一个_config.yml文件，用你喜欢的编辑器打开它，它就是我们的整个站点的配置文件，当然了后面会有一个主题配置文件也叫_config.yml。
    我们可以和清楚的看到，上面有一个#Site，下面的每一个配置文件都有这些相当于每一块的标题，我把它们都叫做什么什么块，你们随意称呼，其实后面还有一堆英文，那些就是一些作用的说明，怎么用，这里只简单的介绍第一个Site块。

站点配置

  • 打开站点配置文件，找到Site块，你可以直接搜索site。我们来看一看他都有什么。

    title: 你的博客标题
    subtitle: 你的博客副标题
    description: 描述
    author: 作者
    language: 你网站用的语言
    timezone: 时区

  • 之前的版本中还有个theme，它代表的是你用到的主题，第一次初始化的化默认的是hexo的主题landscape，当然，这是可以更改的，你可以去hexo主题网站去找到你喜欢的主题。

  • 你要将你喜欢的主题下载到hexo目录下的themes文件下，然后更改站点配置theme的值，为你的主题文件夹的名称，比如我用的next主题。theme: next
    `注意：在更改这些站点配置的值的时候，冒号和值之间要有一个空格，不然会报错的。

创建新的文章

输入

这时候你就可以看见hexo/source/_posts就会有一个新的.md文档，这就是你创建的文章了， 里面还有个默认的文档，你可以保留，也可以删除···

渲染文章

hexo开始渲染你的文章，生成对应的html，css等文件，生成的文件会放在一个叫public文件夹中。

本地浏览

这时hexo会自动开启服务，你可以在信息的最后看到Hexo is running at http://localhost:4000/.Press Ctrl+C to stop.的字样，这个时候你就可以打开浏览器，输入

现在的hexo已经可以实时预览了，当你运行hexo s --debug后，假如你修改了某一个配置项，保存后，命令行窗口会自动发生变化，这是hexo监控你的动作。你改变文章，它会马上做出反应，自动的渲染，相当于同时执行 hexo g和hexo s。

部署到Github上

  • 你写博客当然是给别人看的，所以我们需要部署到Github上。去github注册。
  • 注册后，点击new repository，新建一个仓库。
  • 仓库名称，一定要写成yourname.github.io这是github page 的命名规范，yourname就是你注册时的名称。
  • 修改站点配置，找到Deployment块。

    type: git
    repository: 你的仓库地址，例如https://github.com/yourname/yourname.github.io.git
    branch: 分支名称 一般为master
    message: update
